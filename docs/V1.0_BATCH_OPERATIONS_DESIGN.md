# v1.0.0 - Batch Operations Design Document

## Overview

This document outlines the design for batch operations capabilities in ImHex MCP v1.0.0, enabling automated processing of multiple binary files for large-scale analysis workflows.

## Goals

1. **Directory Processing** - Open and analyze all files in a directory
2. **Batch Pattern Extraction** - Search for patterns across multiple files simultaneously
3. **Report Generation** - Create comprehensive analysis reports
4. **Automation** - Enable scriptable workflows for reverse engineering teams

## Use Cases

### 1. Firmware Version Analysis
**Scenario**: Analyze 50 firmware versions to track security patches

**Workflow**:
```
User: "Analyze all firmware files in ./firmware_releases/ and find which versions contain the security patch at offset 0x8000"

Claude will:
1. Open all .bin files in directory (batch_open_directory)
2. Search each file for patch signature (batch_search)
3. Generate comparison report (batch_report)
4. Identify versions with/without patch
```

### 2. Malware Family Analysis
**Scenario**: Process 200 malware samples to identify common patterns

**Workflow**:
```
User: "Scan ./malware_samples/ for common shellcode patterns and C2 infrastructure"

Claude will:
1. Open all samples (batch_open_directory with filters)
2. Multi-pattern search across all files (batch_search with pattern list)
3. Extract and correlate findings
4. Generate malware family report with IoCs
```

### 3. Binary Diff Campaign
**Scenario**: Compare a reference binary against 100 variants

**Workflow**:
```
User: "Compare reference.exe against all files in ./variants/ and report differences"

Claude will:
1. Open reference file
2. Batch open variants directory
3. Diff each variant against reference (batch_diff)
4. Generate similarity matrix and change report
```

## Architecture

### New Endpoints

#### 1. `batch/open_directory`
**Purpose**: Open multiple files from a directory with filtering

**Parameters**:
```json
{
  "directory": "/path/to/binaries",
  "pattern": "*.bin",           // glob pattern (optional)
  "recursive": false,            // search subdirectories (optional)
  "max_files": 100,              // safety limit (optional, default 100)
  "filters": {                   // optional filters
    "min_size": 1024,            // minimum file size in bytes
    "max_size": 104857600,       // maximum file size (100MB)
    "extensions": [".bin", ".exe", ".elf"]
  }
}
```

**Response**:
```json
{
  "status": "success",
  "data": {
    "opened_files": [
      {"id": 1, "path": "/path/to/file1.bin", "size": 4096},
      {"id": 2, "path": "/path/to/file2.bin", "size": 8192}
    ],
    "total_opened": 42,
    "skipped": 3,
    "errors": []
  }
}
```

#### 2. `batch/search`
**Purpose**: Search for patterns across all open files or specified subset

**Parameters**:
```json
{
  "patterns": [
    {"type": "hex", "value": "4D5A"},
    {"type": "string", "value": "http://"},
    {"type": "regex", "value": "[0-9]{1,3}\\.[0-9]{1,3}\\.[0-9]{1,3}\\.[0-9]{1,3}"}
  ],
  "provider_ids": [1, 2, 3],  // optional, defaults to all open files
  "max_matches_per_file": 1000,
  "export_format": "json"      // or "csv"
}
```

**Response**:
```json
{
  "status": "success",
  "data": {
    "results": [
      {
        "provider_id": 1,
        "file": "/path/to/file1.bin",
        "patterns": [
          {
            "pattern": "4D5A",
            "matches": [
              {"offset": 0, "context": "4D5A90000300..."}
            ]
          }
        ],
        "total_matches": 1
      }
    ],
    "summary": {
      "files_searched": 42,
      "total_matches": 156,
      "patterns_found": 3
    }
  }
}
```

#### 3. `batch/hash`
**Purpose**: Calculate hashes for all open files

**Parameters**:
```json
{
  "algorithms": ["md5", "sha256"],
  "provider_ids": [1, 2, 3],  // optional
  "export_format": "json"      // or "csv"
}
```

**Response**:
```json
{
  "status": "success",
  "data": {
    "hashes": [
      {
        "provider_id": 1,
        "file": "/path/to/file1.bin",
        "size": 4096,
        "md5": "d41d8cd98f00b204e9800998ecf8427e",
        "sha256": "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855"
      }
    ],
    "total_files": 42
  }
}
```

#### 4. `batch/diff`
**Purpose**: Compare a reference file against multiple targets

**Parameters**:
```json
{
  "reference_id": 1,
  "target_ids": [2, 3, 4],    // or "all" for all other open files
  "algorithm": "myers",
  "max_diff_regions": 1000
}
```

**Response**:
```json
{
  "status": "success",
  "data": {
    "diffs": [
      {
        "target_id": 2,
        "target_file": "/path/to/variant1.bin",
        "similarity": 95.5,
        "diff_regions": 12,
        "regions": [...]
      }
    ],
    "summary": {
      "files_compared": 3,
      "avg_similarity": 94.2,
      "most_similar": 2,
      "least_similar": 4
    }
  }
}
```

#### 5. `batch/report`
**Purpose**: Generate comprehensive analysis report

**Parameters**:
```json
{
  "format": "markdown",  // or "json", "html"
  "include": {
    "file_list": true,
    "hashes": true,
    "search_results": true,
    "diff_summary": true,
    "statistics": true
  },
  "output_path": "/path/to/report.md"  // optional, returns content if not specified
}
```

**Response**:
```json
{
  "status": "success",
  "data": {
    "report_content": "# Binary Analysis Report\n\n...",
    "saved_to": "/path/to/report.md",
    "size": 45678
  }
}
```

#### 6. `batch/close_all`
**Purpose**: Close all open files or filtered subset

**Parameters**:
```json
{
  "filter": {
    "pattern": "*.tmp",     // optional glob
    "directory": "/path"    // optional directory filter
  }
}
```

**Response**:
```json
{
  "status": "success",
  "data": {
    "closed_count": 42,
    "remaining_count": 0
  }
}
```

## MCP Tools

### `imhex_batch_open_directory`
**Description**: Open all binary files from a directory for batch analysis

**Parameters**:
- `directory` (required): Directory path
- `pattern` (optional): Glob pattern (e.g., "*.bin")
- `recursive` (optional): Search subdirectories
- `max_files` (optional): Maximum files to open (default 100)

### `imhex_batch_search`
**Description**: Search for patterns across all open files

**Parameters**:
- `patterns` (required): List of patterns to search
- `provider_ids` (optional): Specific files to search
- `export_format` (optional): Output format (json/csv)

### `imhex_batch_hash`
**Description**: Calculate hashes for multiple files

**Parameters**:
- `algorithms` (required): List of hash algorithms
- `provider_ids` (optional): Specific files to hash
- `export_format` (optional): Output format (json/csv)

### `imhex_batch_diff`
**Description**: Compare reference file against multiple targets

**Parameters**:
- `reference_id` (required): Reference file provider ID
- `target_ids` (required): Target file provider IDs or "all"
- `algorithm` (optional): Diff algorithm (default "myers")

### `imhex_batch_report`
**Description**: Generate comprehensive analysis report

**Parameters**:
- `format` (required): Report format (markdown/json/html)
- `include` (optional): Sections to include
- `output_path` (optional): Save report to file

### `imhex_batch_close_all`
**Description**: Close all open files or filtered subset

**Parameters**:
- `filter` (optional): Filter criteria (pattern, directory)

## Performance Considerations

### File Limits
- Default max: 100 files per batch_open_directory
- Configurable via parameter
- Warning when approaching memory limits

### Search Optimization
- Chunk-based file reading (1MB chunks)
- Parallel search where applicable
- Result streaming for large datasets

### Memory Management
- Close unused files automatically
- Implement file handle pooling
- Monitor memory usage and warn user

### Progress Reporting
- For long operations (>5 seconds), report progress
- Estimate time remaining
- Allow cancellation via separate endpoint
